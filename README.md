# Trauma Note Wellness App

Non-clinical wellness journaling and self-reflection UI experiments for Japanese users.

This repository is a working prototype for AI-assisted note-taking around sensitive personal reflection. It is not a medical device, diagnostic system, therapy product, or crisis-support tool. The goal is to document and improve safer interface patterns for personal journaling, user-controlled saving, progress tracking, and careful AI-assisted reflection.

## What This Project Explores

- Japanese-first wellness journaling UI
- User-controlled note saving and deletion
- Clear boundaries between temporary AI chat and persistent notes
- Diagnosis-history and progress-tracking interface patterns
- Feedback capture for iterative UX improvement
- Firebase-backed prototype deployment patterns

## Safety Framing

This project should not be used to diagnose, treat, or make clinical decisions. Any AI-generated output should be treated as reflective support only. For medical, mental-health, emergency, or crisis situations, users should seek qualified professional help or local emergency support.

## Why Open Source

Sensitive AI-assisted journaling apps need better public examples for privacy, persistence, consent, and user agency. This project is being opened so builders can review, improve, and adapt safer UX patterns for non-clinical self-reflection tools.

## Maintenance Roadmap

- Audit Firebase configuration and data boundaries
- Separate public demo configuration from private production settings
- Add automated checks for accidental secrets
- Add accessibility review
- Improve Japanese and English documentation
- Add privacy and safety notes in the UI
- Break large HTML prototypes into maintainable modules
- Add issue templates for safety, privacy, and UX reports
